Scope of var, let, and const

We all devs know that there are many upgraded feature comes with ES6, Develepers are become familiar with these amazing features....
Scope means where these (var,let,const) variables are available for use.

var

var is a globally scoped variable. The scope is global when it's declared without or outside of a function. That means any kind of var variable available for use in whole window. But if it's in a function then we can access only within that function, because var is function scoped...

let

let is block scope. Block means codes in a {} curly braces. Any code in curly braces is block. So variable declare with let in a block, will only work in that block not outside the block. let variable values are can be updated but not redeclared...

const

Variable declaring with const mean maintaining constant values. const is also a block scope like let. let and const have some similarity, but const can not be updated and redeclared. That means if we declare any value with const it can not be change or redeclared...

The use cases of null and undefined

Undefined means variable is declared but the value is not assigned or value is not valid, And Null means declared variable have no value or the value is empty.

Undefined

Undefined means not defined When we declare a variable but we dont put any value or the value is not valid, it will return undefined. Also if we create a function but do not pass the arguments for parameters of the default value taken as undefined....

Null

Null means having no value. Null is an assigment value to a variable. The variable value with null means contains no value. When we assign null as a value to any variable, that means its empty or blank. Null is an object in JavaScript...

WHAT IS A REST API ?

A REST API is an API that conforms to the design principles of the REST, or representational state transfer architectural style. REST API is a way of accessing web services more simple and flexible way without having any processing.

A request is sent from client to server in the form of a web URL as HTTP GET or POST or PUT or DELETE request. After that, a response comes back from the server in the form of a resource which can be anything like HTML, XML, Image, or JSON. But now JSON is the most popular format being used in Web Services.

In HTTP there are five methods that are commonly used in a REST-based Architecture i.e., POST, GET, PUT, PATCH, and DELETE. These correspond to create, read, update, and delete (or CRUD) operations respectively. There are other methods which are less frequently used like OPTIONS and HEAD.
